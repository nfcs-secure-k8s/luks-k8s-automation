# Automated Encryption of Kubernetes Block Storage for Trusted Research Environment

For researchers carrying out experiments on sensitive data, there is a clear need to ensure that the volumes storing that data are encrypted at rest. In OpenStack, the existing approach typically relies on Cinder volume encryption with Barbican for key management. However, in this model, key management remains under the control of the hosting administrator. It is also not straightforward to provide strong per-project, per-institution, or research-domain-specific key isolation, because native Cinder encryption is generally implemented at the infrastructure level rather than being designed around independent tenant-controlled key ownership.

For projects involving multiple institutions, where each institution must retain exclusive control of its own encryption keys, this approach may not be sufficient. In such cases, a more tailored solution is needed to support institution-managed or researcher-managed keys in a bring-your-own-key style model. For this reason, this project explores a custom Kubernetes-based solution using a Kubernetes Operator built with Kopf and integrated with HashiCorp Vault for key management.

The benefit of this approach is that it enables stronger separation of responsibilities between the infrastructure provider and the data owner. Each institution can manage its own keys independently, improving governance, supporting federated trust models, and aligning more closely with regulatory and organisational requirements for control over sensitive data. It also provides greater flexibility for enforcing project-specific encryption policies within Kubernetes workloads, while still operating on top of OpenStack-backed storage.
